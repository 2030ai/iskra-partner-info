# Варианты развёртывания

Дата обновления: 2026-06-30

## Cloud

Cloud подходит для быстрого старта, демонстраций, пилотов и команд, которым важна скорость запуска. Условия по данным, моделям, инфраструктуре и поддержке зависят от выбранного предложения.

## Российский контур

Российский контур обсуждается для клиентов, которым важно размещение и обработка в российской инфраструктуре. Конкретные условия фиксируются в спецификации и техническом описании.

## On-premise

On-premise подходит клиентам с повышенными требованиями к инфраструктуре, данным, LLM, gateway, доступам и эксплуатации. Такой запуск требует обследования и отдельного проекта внедрения.

## Headless и интеграционный формат

В отдельных проектах Искра может обсуждаться как часть интеграционного контура: навыки, MCP, внутренние инструменты, корпоративные процессы и специальные интерфейсы.

## Как говорить про данные

- Для обычного cloud нельзя обещать, что данные никогда не передаются во внешние сервисы.
- Для российского контура нужно указывать конкретные условия выбранной инфраструктуры.
- Для fully local on-premise без внешнего LLM/gateway внешняя передача данных может быть исключена архитектурно после проверки конфигурации.
- Любая формулировка про данные должна быть согласована с договором, спецификацией и technical scope.

## Как говорить про LLM

- Искре нужен доступ к LLM: через внешний API, gateway или локальные модели в выбранном контуре.
- В cloud и интеграционных сценариях состав провайдеров зависит от текущей конфигурации продукта и коммерческого предложения.
- В on-premise проекте LLM, gateway, GPU, серверы и эксплуатация считаются отдельными элементами, если они прямо не включены в спецификацию.
